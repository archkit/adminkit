# adminkit チームレビュー: 問題点

5 名のチームメンバーが指摘した課題をまとめる。

---

## モバイル UX

- ~~テーブルの横スクロールが可能であることがユーザーに伝わらない~~ → `c-table-scroll` にグラデーションシャドウヒントを追加済み
- コンテンツ padding の縮小と、サイドバーのドロワー化だけで、モバイル特化の体験設計がない
- フォームのステップ分割やフローティングラベルなど、モバイル入力を積極的に改善する仕組みがない

## ~~フォームバリデーション~~ → 対応済み

- ~~`:invalid` のスタイルが `border-color: --status-fail` のみ~~ → `:user-invalid` による段階的エラー表示を追加済み
- ~~`:user-invalid` と組み合わせた、入力時点での段階的なエラー表示がない~~ → `.error-message` クラスで `:user-invalid` 連動表示を実装済み
- エラーメッセージの表示/非表示の連動、エラー箇所へのスクロール、サマリー表示の仕組みがフレームワーク側にない（利用者側の実装）
- ~~送信後ではなく入力中にフィードバックが欲しい~~ → `:user-invalid` で対応済み

## ~~キーボードナビゲーション~~ → 対応済み

- ~~タブコンポーネント: 矢印キーでのタブ移動が JS に未実装~~ → WAI-ARIA Tabs パターン準拠で実装済み（矢印キー + Home/End + tabindex 管理）
- ~~ドロップダウン: メニューアイテム間の矢印キー移動がない~~ → WAI-ARIA Menu パターン準拠で実装済み（矢印キー + 自動フォーカス）

## ~~アクセシビリティの細部~~ → 部分対応

- ~~`<small>` によるヘルパーテキストが `aria-describedby` で input に紐づいていない~~ → デモページに `aria-describedby` を追加済み
- テーブルのソートアイコン（▲▼）にテキスト代替がない

## ステータス色の混色率

- alert: 8%, banner: 12%, toast: 別計算 — コンポーネントごとに背景の混色率が異なる
- `--status-bg-subtle` のような統一トークンがないため、バリアント追加時に判断基準が不明確

## ~~CDN 依存~~ → 対応済み

- ~~Lucide を esm.sh から読んでおり、CDN 障害時にアイコンが全て消える~~
- esbuild で Lucide をローカルバンドル済み（`dist/adminkit.min.js`）。CDN 不要

## テスト・品質保証

- ビジュアルリグレッションテスト、stylelint、axe 等の自動テストの仕組みがない
- テーマ × モード × バリアントの組み合わせ爆発を人力で確認するのは限界がある
- コンポーネントギャラリーが事実上の手動テストスイート

## ミニサイドバーの制約

- 縦スクロール非対応。メニュー項目が多い場合にフルサイドバーに頼るしかない
- メニュー検索機能がない（エンドユーザーの声）

## CSS コード分割

- 全コンポーネントが単一ファイルにバンドルされ、ページ単位のコード分割ができない
- 管理画面規模では実害は小さいが、拡張時の検討事項
